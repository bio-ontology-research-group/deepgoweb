DeepGOWeb function prediction webserver

DeepGOWeb is a webserver for protein function prediction. It offers two models: DeepGOPlus (a convolutional sequence model combined with DIAMOND homology transfer) and DeepGO-PlusPlus-Light, a CPU-only multi-evidence ensemble (DIAMOND, STRING-Net, a hierarchy-aware CNN, ESM2 embeddings and ProteInfer combined by a learned integrator). Predictions are available in three ways. First, using the Prediction web page, users can submit protein sequences and obtain predictions which are downloadable in JSON/CSV format. Second, DeepGOWeb provides a REST API for accessing the service programmatically. Finally, users can use our SPARQL endpoint to call the predictors within a SPARQL query.

Web page for submitting protein sequences

Users should provide the following data to use the service on the Prediction page:

  • Format: FASTA format or Raw Sequences separated by a newline.
  • Threshold: a value between 0.1 and 1.0 for filtering predictions by the confidence score of the model.
  • Data: Protein sequences in the selected format. A maximum of 10 sequences are allowed in one request.

Additional options:

  • Prediction model: choose DeepGOPlus (CNN + DIAMOND) or DeepGO-PlusPlus-Light, the CPU multi-evidence ensemble. For the latter, the result page also shows each individual predictor's output under "Individual predictor outputs".
  • Show only the most specific terms (on by default): contracts each aspect to the most specific predicted GO terms, hiding ancestors above the threshold that are implied by a more specific descendant. The full propagated set is always available in the JSON/CSV export.
Obsolete GO terms in a prediction are automatically transferred to their replacement (via the GO replaced_by/consider fields), or flagged, rather than dropped.

After submitting the request, users will be redirected to results page where they can see the predictions and download them in JSON format. Users can also save the link to the results page and come back to it anytime.

Output description

The output page provides function predictions for each protein. A prediction includes GO terms for the Biological Process, Molecular Function, and Cellular Component sub-ontologies with confidence scores. Only predictions with a confidence above the threshold parameter are shown. We also show a list of proteins that are used for predictions based on sequence similarity and their similarity score to the query protein. The results are provided under the CC-BY License and can be downloaded in JSON/CSV formats. Here is the example output page.

Choosing a model and getting component outputs

A second, backwards-compatible endpoint /deepgo/api/predict takes an extra predictor field. Use "deepgoplus" (the default) for the model above, or "dgpp-light" for the CPU-only DeepGO-PlusPlus-Light ensemble. For dgpp-light the response additionally includes a components field with each individual predictor's own top GO terms (DIAMOND, STRING-Net, the hierarchy-aware CNN, the ESM2 embedding kNN and head, and ProteInfer).

import requests

r = requests.post('http://deepgoplus.bio2vec.net/deepgo/api/predict',
                  json={'predictor': 'dgpp-light',
                        'data_format': 'enter',
                        'data': sequence,
                        'threshold': 0.1})
result = r.json()
integrated = result['predictions'][0]['functions']   # combined GO predictions
components = result['components'][0]                   # {component_label: [[go_id, name, score], ...]}

The original /deepgo/api/create endpoint is unchanged; clients that do not send predictor keep the original behaviour.

SPARQL

The SPARQL endpoint allows to call the DeepGOPlus function prediction model within a SPARQL query. We provide a custom SPARQL function called "deepgo" which takes a protein sequence and prediction threshold as an input and returns the predicted functions along with the subontology, label, and prediction score. The output can be downloaded in different formats such as json, xml, csv or text.

Two additional property functions let you select the prediction model and inspect its internals (the original dg:deepgo function is unchanged):

  • dg:predict(?sequence ?threshold ?predictor) — like dg:deepgo but with an extra predictor argument, e.g. "dgpp-light" (the CPU DeepGO-PlusPlus-Light ensemble) or "deepgoplus" (default). Binds (?subontology ?go ?label ?score).
  • dg:components(?sequence ?threshold ?predictor) — returns each individual predictor substream (DIAMOND, STRING-Net, the hierarchy-aware CNN, ESM2 kNN and head, ProteInfer) separately. The first bound variable is the component name: (?component ?go ?label ?score).

Commandline tool

Users can also install DeepGOPlus on their system and run DeepGOPlus locally. This is useful when prediction functions for a large amount of protein sequences.

Installation

pip

pip install deepgoplus
Download the data

Download all the files from http://deepgoplus.bio2vec.net/data/data.tar.gz and extract them into data folder

Run:

deepgoplus --data-root path_to_data_folder --in-file input_fasta_filename

Prediction Model

We use DeepGOPlus method to predict protein functions. The method is based on combination of Convolutional Neural Network (CNN) model and sequence similarity.

DeepGOPlus CNN Model

The figure describes the architecture of our deep learning model. First, the input sequence is converted to a one-hot encoded representation of size 21 × 2000, where a one-hot vector of length 21 represents an amino acid (AA) and 2000 is the input length. Sequences with a length less than 2000 are padded with zeros and longer sequences are split into smaller chunks with less than 2000 AAs. This input is passed to a set of CNN layers with different filter sizes of 8, 16, …, 128. Each of the CNN layers has 512 filters which learn specific sequence motifs of a particular size. Each filter is scanning the sequence and their maximum score is pooled using a MaxPooling layer. In total, we generate a feature vector of size 8192 where each value represents a score that indicates the presence of a relevant sequence motif. This vector is passed to the fully connected classification layer which outputs the predictions

DeepGO-PlusPlus-Light

DeepGO-PlusPlus-Light is a CPU-only, multi-evidence ensemble aimed at generalising to novel proteins. It combines several heterogeneous components with a learned per-aspect logistic-regression integrator:

  • DIAMOND BLAST-KNN — GO transfer from sequence homologs.
  • STRING Net-KNN — a homology-bridged protein-network signal.
  • Hierarchy-aware CNN — a 1D sequence CNN trained with the C-HMCNN Max-Constraint Module over the GO is_a ∪ part_of DAG.
  • ESM2-35M embedding kNN — nearest-neighbour GO transfer over ESM2 protein language-model embeddings.
  • ESM2-35M hierarchy-aware head — a parametric per-aspect head over the same ESM2 embedding, also trained with the Max-Constraint Module.
  • ProteInfer — a sequence CNN GO predictor.

Because every component runs on CPU, the model is served without a GPU. The result page exposes each component's own top predictions in an "Individual predictor outputs" section, and the same substreams are available through the /deepgo/api/predict endpoint and the dg:components SPARQL function.
